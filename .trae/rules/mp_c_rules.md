# 🐾 C 端小程序（宠物主端）业务愿景与功能规则

本规则面向 **宠物主（C 端用户）**，用于从业务角度定义小程序的价值主张、核心路径与功能边界，帮助设计与开发始终围绕“下单成功与服务放心”展开。

## 1. 产品愿景
让每一次上门服务，都像“把熟悉的人请回家”一样安心：下单简单、过程透明、结果可控。

### 1.1 价值主张
- **更省心**：从选择到预约再到履约，全程少决策、少沟通。
- **更放心**：服务人员可靠可见，过程可追踪、可举证。
- **更确定**：价格结构清晰，时间与到达预期明确。

## 2. 核心用户画像
### 2.1 新手宠物主
首次下单最担心“人靠不靠谱、会不会踩坑、怎么选”。

### 2.2 高频需求宠物主
有稳定需求（出差喂猫、固定遛狗、周期洗护），关注“效率、复购与一致性”。

### 2.3 精细化养宠用户
对服务细节要求高，关注“服务方案、加项透明、过程记录质量”。

## 3. C 端关键路径（必须顺滑）
### 3.1 首单路径（转化）
1) 选择服务类型 → 2) 选择时间与地址 → 3) 浏览服务人员列表 → 4) 进入详情页建立信任 → 5) 选择规格/加项 → 6) 确认订单与支付 → 7) 预约成功。

### 3.2 履约路径（信任）
1) 接单确认 → 2) 服务人员出发/到达 → 3) 服务中过程记录可见 → 4) 服务完成确认 → 5) 评价与售后。

### 3.3 复购路径（增长）
1) 收藏/关注服务人员 → 2) 一键复购（同地址同宠物）→ 3) 常用服务与常用时间段 → 4) 会员/优惠与提醒。

## 4. 功能域划分（C 端必须聚焦）
### 4.1 首页与发现
- 快速入口：上门洗护/上门喂猫/遛狗
- 场景化推荐：出差喂猫、固定遛狗、换季洗护
- 运营位：限时补贴、专题服务、新人首单

### 4.2 搜索与筛选（降低选择成本）
- 位置与可达：服务半径、到达预期
- 时间可用：可预约时段清晰展示
- 信任信号：评分、服务次数、认证、擅长宠种
- 价格与规格：基础价与常用加项可对比

### 4.3 服务人员详情页（建立信任的主战场）
- 个人介绍与能力标签
- 服务项目与价格结构（基础 + 加项 + 规则）
- 评价体系：结构化标签 + 真实图文（强调可读性）
- 服务案例：作品集/过程记录样例

### 4.4 下单与预约（减少犹豫）
- 地址管理：常用地址、门禁/入户信息的安全存储与提示
- 宠物档案：多宠、性格、禁忌、注意事项
- 订单确认：费用明细、时间、服务范围、取消规则
- 支付：支付成功状态明确，失败可恢复

### 4.5 订单与履约追踪（透明与确定性）
- 状态流转：已预约/已接单/进行中/已完成/售后中
- 关键节点：出发、到达、开始、结束（时间戳）
- 过程记录：按服务类型聚合展示（图/视频/文本）
- 联系方式：优先站内消息与安全通道，减少隐私泄露

### 4.6 评价与售后（保护体验底线）
- 评价：评分 + 标签 + 可选图文
- 售后：问题类型选择、举证上传、处理进度
- 规则透明：取消/改期/迟到/加项争议处理

### 4.7 我的（资产沉淀）
- 收藏与关注：服务人员、常用服务
- 优惠与会员：券包、订阅、活动
- 安全与隐私：地址与入户信息管理、授权说明

## 5. C 端业务约束（Do / Don’t）
### 5.1 必须坚持
- **信任优先**：一切信息组织都应服务“是否敢下单”。
- **明细优先**：价格与服务范围必须可解释、可对比、可追溯。
- **少输入**：尽量用档案化（宠物档案/地址模板）替代自由填写。
- **强确定性**：时间、到达、费用、规则都应给出明确预期。

### 5.2 避免发生
- 让用户在关键决策点看到模糊表达（例如“到时再说”“可能加价”）。
- 用复杂的筛选项堆满屏幕，导致“越选越不知道选谁”。
- 把履约过程变成聊天记录，缺少结构化的过程凭证。

## 6. 服务类型差异（用于信息与过程展示）
### 6.1 上门喂猫
- 关注点：是否到场、是否按要求执行、多猫区分、环境整洁
- 最有价值的记录：到达自拍/门牌、喂食与换水、猫砂与卫生、互动片段

### 6.2 遛狗
- 关注点：安全、时长、路线、突发应对
- 最有价值的记录：开始/结束时间、路线或定位片段、排泄与清洁说明

### 6.3 上门洗护
- 关注点：专业度、耗时预期、加项透明、宠物应激与安全
- 最有价值的记录：前后对比、关键步骤（吹干/修剪）、加项确认依据

## 7. 关键指标（C 端产品成败衡量）
- 首单转化率、下单完成率、支付成功率
- 取消/改期率、售后发起率、客服介入率
- 履约满意度（评分、差评率）、复购率
- 收藏/关注转化、常用档案创建率

## 8. 中长期方向（设计时预留）
- 复购与订阅：固定遛狗排期、出差喂猫计划
- 多宠家庭：更强的宠物档案与分宠记录
- 安全体系：更精细的认证分层、保险与保障说明
